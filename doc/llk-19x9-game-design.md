# 19x9 连连看玩法设计文档

## 1. 玩法目标

本玩法是在地砖区域上实现一个标准连连看游戏。玩家在 19 列、9 行的棋盘中点击两个相同图案的牌块；如果两张牌可以通过不超过两次转折的连线连接，则这两张牌消除。所有可消除牌块清空后，游戏胜利。

当前棋盘用于适配 19:9 的游戏画面比例，单个格子在世界坐标中的尺寸为 `2.5 x 2.5`。

## 2. 棋盘规格

- 列数：`19`
- 行数：`9`
- 单格尺寸：`2.5 x 2.5`
- 总格子数：`19 * 9 = 171`
- 棋盘世界尺寸：
  - X 方向宽度：`19 * 2.5 = 47.5`
  - Z 方向高度：`9 * 2.5 = 22.5`
- 当前棋盘中心：
  - X：`-23.2519999999`
  - Z：`0.25`
- 棋盘边界：
  - `xMin = centerX - 47.5 / 2`
  - `xMax = centerX + 47.5 / 2`
  - `zMin = centerZ - 22.5 / 2`
  - `zMax = centerZ + 22.5 / 2`

注意：171 是奇数，而连连看需要成对消除。因此初始布局默认使用 `170` 个有效牌格和 `1` 个空格。空格值记为 `0`，不参与点击和消除。

## 3. 数据结构

棋盘使用二维数组保存，数组结构为 `rows x columns`，即 `9 x 19`。

单个格子数据建议如下：

```ts
type LlkGridCell = {
  row: number
  column: number
  x: number
  z: number
  value: number
  matched: boolean
  tileUnit?: any
  buttonNode?: any
  stickerNode?: any
}
```

字段说明：

- `row`：行号，范围 `0..8`
- `column`：列号，范围 `0..18`
- `x`：格子中心点世界 X 坐标
- `z`：格子中心点世界 Z 坐标
- `value`：牌面类型，`0` 表示空格，`1..10` 表示 10 种图案
- `matched`：是否已经被消除
- `tileUnit`：运行时生成的实体方块，可选
- `buttonNode`：对应的 UI 按钮节点，可选
- `stickerNode`：对应的贴纸或图案节点，可选

## 4. 初始牌面生成

初始牌面需要满足：

- 有效牌数量为偶数，默认 `170`
- 每个图案成对出现
- 图案种类默认 `10` 种
- 生成后随机洗牌
- 预留 `1` 个空格，值为 `0`

推荐生成流程：

1. 创建空数组 `values`
2. 计算有效牌数量：`validCellCount = 19 * 9 - 1 = 170`
3. 计算牌对数量：`pairCount = validCellCount / 2 = 85`
4. 循环生成 85 对牌：
   - 第 `i` 对的值为 `(i % 10) + 1`
   - 向 `values` 加入两个相同值
5. 加入一个 `0` 作为空格
6. 使用随机洗牌打散 `values`
7. 按行列填入 `9 x 19` 二维数组

## 5. 坐标映射

每个格子的中心点坐标：

```ts
x = xMin + cellSize * (column + 0.5)
z = zMin + cellSize * (row + 0.5)
```

其中：

- `cellSize = 2.5`
- `column` 范围 `0..18`
- `row` 范围 `0..8`

UI 坐标和世界格子需要保持一一绑定。推荐将二维数组作为唯一数据源，所有 UI、贴纸、实体方块都从数组生成并回写引用。

## 6. 牌块表现

每个有效格子应该由三层组成：

- 交互层：玩家点击用的按钮或碰撞体
- 背景层：牌块底板
- 图案层：贴纸或图片，按 `value` 显示对应图案

空格 `value = 0`：

- 不显示牌块
- 不可点击
- 连线检测时视为空路

已消除格子：

- `matched = true`
- `value` 可保持原值，也可以置为 `0`
- UI 和实体需要隐藏或销毁
- 连线检测时视为空路

## 7. 点击流程

玩家点击一个格子的流程：

1. 根据点击对象找到对应 `row, column`
2. 读取 `grid[row][column]`
3. 如果 `value = 0` 或 `matched = true`，忽略
4. 如果当前没有已选牌：
   - 记录为第一张牌
   - 显示选中效果
5. 如果当前已有第一张牌：
   - 如果点击同一格，取消选中
   - 如果两张牌 `value` 不同，切换选中或提示错误
   - 如果两张牌 `value` 相同，执行连线检测
6. 连线检测成功：
   - 播放连线表现
   - 消除两张牌
   - 检查胜利
7. 连线检测失败：
   - 保留或取消选中，按体验决定

## 8. 连线规则

标准连连看规则：两张相同牌之间可以通过最多 `2` 次转折连接。

可用路径类型：

- 0 折：两点同行或同列，中间无阻挡
- 1 折：存在一个拐点，形成两段直线，两段都无阻挡
- 2 折：存在两个拐点，形成三段直线，三段都无阻挡

路径可以经过：

- 空格
- 已消除格
- 棋盘外一圈虚拟空格

路径不能经过：

- 未消除的有效牌格

## 9. 连线检测算法

推荐在二维数组外扩一圈空边界，形成 `11 x 21` 的检测网格：

- 原棋盘行列映射到外扩网格时，行列各加 `1`
- 外圈全部视为空
- 这样允许从棋盘外绕线

判断两格是否可连接：

1. 两格必须不同
2. 两格值必须相同
3. 两格都未消除
4. 使用 BFS 搜索路径
5. BFS 状态包含：
   - 当前行
   - 当前列
   - 当前方向
   - 已转折次数
6. 转折次数超过 `2` 时丢弃
7. 到达目标格时成功

方向定义：

```ts
const directions = [
  { dr: -1, dc: 0 },
  { dr: 1, dc: 0 },
  { dr: 0, dc: -1 },
  { dr: 0, dc: 1 },
]
```

## 10. 消除逻辑

消除成功时：

- 设置两个格子的 `matched = true`
- 隐藏或销毁对应 UI
- 隐藏或销毁对应贴纸
- 隐藏或销毁对应实体方块
- 清空当前选中状态
- 播放连线、消失、音效反馈

为了简化路径检测，消除后可以同时把 `value` 置为 `0`。如果后续需要统计原始牌面，可以保留 `value`，只用 `matched` 判断是否为空。

## 11. 胜利条件

游戏胜利条件：

- 所有有效牌格都已消除
- 即不存在 `value > 0 && matched === false` 的格子

胜利后：

- 停止计时
- 显示胜利 UI
- 禁止继续点击
- 可提供重新开始按钮

## 12. 无解处理

由于随机生成可能出现无解局面，需要设计处理方式。

推荐方案：

- 每次消除后扫描是否存在可连接牌对
- 如果不存在可连接牌对，但仍有剩余牌：
  - 自动洗牌剩余未消除牌
  - 或提供“洗牌”按钮

洗牌规则：

- 只收集未消除且 `value > 0` 的格子值
- 随机打乱后重新填回这些格子
- 保留空格和已消除格

## 13. UI 与数组绑定

每个生成出来的显示对象必须能反查数组坐标。

推荐命名规则：

```text
LLK_TILE_{row}_{column}
LLK_BUTTON_{row}_{column}
LLK_STICKER_{row}_{column}
```

示例：

```text
LLK_TILE_03_12
LLK_BUTTON_03_12
LLK_STICKER_03_12
```

点击时可以通过节点名解析行列，也可以在创建时维护映射表：

```ts
const nodeToCell: Record<string, { row: number; column: number }> = {}
```

## 16. 验收标准

基础验收：

- 棋盘在画面内完整显示
- 19x9 坐标映射正确
- 单格世界尺寸为 `2.5 x 2.5`
- 初始有效牌数量为偶数
- 每个有效牌值都能找到对应配对
- 空格不显示、不响应点击

玩法验收：

- 相同牌且路径合法时可以消除
- 相同牌但路径被阻挡时不能消除
- 不同牌不能消除
- 消除后路径可以穿过已消除位置
- 所有牌消除后触发胜利

表现验收：

- 不显示调试数字
- 不显示四角定位方块
- 不显示残留白色 UI 板
- 点击反馈清晰
- 消除反馈清晰
